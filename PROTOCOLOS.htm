PROTOCOLOS 

En esta exposición se abordarán los protocolos TCP/IP y FTP, pilares fundamentales de las redes informáticas y del funcionamiento de Internet. Se explicará su función principal, cómo permiten la comunicación entre distintos dispositivos y su importancia en la transmisión segura y eficiente de datos. Además, se analizará el papel que desempeñan estos protocolos en el intercambio de información dentro de las redes modernas.

-TCP/IP

Los protocolos son conjuntos de normas que permiten que las máquinas y los programas se comuniquen entre sí intercambiando información de forma correcta. Para que la comunicación funcione, todos los dispositivos implicados deben seguir las mismas reglas. El conjunto de protocolos TCP/IP se organiza en capas y es la base de la comunicación en Internet.

El TCP (Transmission Control Protocol) pertenece a la capa de transporte y es un protocolo fiable y orientado a conexión. Su función es asegurar que la información enviada desde una aplicación llegue completa y en orden al destino. Si se pierde algún paquete o hay errores, TCP solicita su reenvío y solo entrega los datos al nivel de aplicación cuando están completos.

El IP (Internet Protocol) pertenece a la capa de red y se encarga de enviar los paquetes de datos, llamados datagramas, desde el origen hasta el destino siguiendo la mejor ruta posible. No es un protocolo fiable ni orientado a conexión, ya que no garantiza que los paquetes lleguen todos ni en el mismo orden.

Al trabajar juntos, TCP e IP permiten una comunicación fiable entre dispositivos: IP se encarga de transportar los paquetes y TCP asegura que la información llegue completa al destino.

-FTP

FTP (Protocolo de transferencia de archivos) es un protocolo de red estándar utilizado para la transferencia de archivos de un host a otro a través de una red basada en TCP, como Internet.

 

El FTP funciona abriendo dos conexiones que vinculan las computadoras que intentan comunicarse entre sí. Se designa una conexión para los comandos y respuestas que se envían entre los dos clientes, y el otro canal maneja la transferencia de datos. Durante una transmisión FTP, hay cuatro comandos utilizados por las computadoras, servidores o servidores proxy que se están comunicando. Estos son “enviar”, “obtener”, “cambiar directorio” y “transferir”.

 

Al transferir archivos, FTP utiliza tres modos diferentes: bloquear, transmitir y comprimir. El modo de flujo permite a FTP administrar información en una cadena de datos sin límites entre ellos. El modo de bloqueo separa los datos en bloques y, en el modo de compresión, FTP utiliza un algoritmo llamado Lempel-Ziv para comprimir los datos.

Una de las principales razones por las que las empresas y personas modernas necesitan FTP es su capacidad para realizar grandes transferencias de tamaño de archivo. Al enviar un archivo relativamente pequeño, como un documento de Word, la mayoría de los métodos lo harán, pero  conFTP, puede enviar cientos de gigabytes a la vez y aún así obtener una transmisión fluida.

 

La capacidad de enviar mayores cantidades de datos, a su vez, mejora el flujo de trabajo. Debido a que FTP le permite enviar varios archivos a la vez, puede seleccionar varios y luego enviarlos todos al mismo tiempo. Sin servicios FTP, es posible que tenga que enviarlos uno por uno, cuando podría estar realizando otro trabajo.

 

Por ejemplo, si tiene que transferir una gran colección de documentos importantes de la sede central a una oficina satélite, pero tiene que asistir a una reunión en cinco minutos, puede usar FTP para enviarlos todos a la vez. Incluso si la transferencia tarda 15 minutos en completarse, el FTP puede manejarla, lo que lo libera para asistir a la reunión.

 

Aunque el protocolo de transferencia de hipertexto (HTTP) y el FTP son similares en cuanto a que son protocolos de capa de aplicación que le permiten enviar archivos entre sistemas, existen algunas diferencias claves.

 

HTTP puede admitir varias sesiones al mismo tiempo porque es un protocolo sin estado. Esto significa que no guarda los datos utilizados en una sesión para emplearlos en la siguiente. 

 

El FTP, por otro lado, tiene estado, lo que significa que recopila datos sobre el cliente y los utiliza en la siguiente solicitud que el cliente realiza. Debido a que FTP realiza esta función, está limitada en la cantidad de sesiones que puede admitir simultáneamente.

 

Independientemente del ancho de banda de una red, HTTP tiene el potencial de ser un método de transmisión de datos mucho más eficiente.

Otra diferencia clave es que con el protocolo de transferencia de archivos (FTP), debe haber autenticación de cliente antes de transferir la información. Con HTTP, no se necesita autenticación de cliente.

 

 HTTP utiliza un puerto común y conocido, lo que facilita el trabajo de los firewalls. En algunos casos, el FTP puede ser más difícil de administrar para un firewall.
